Generate a multitenant rails 8 webapp for running 50/50 raffles.

## Core Functionality

# Organization
A user at an organization should be able to register and create a new tenant. That initial user should be the administrator of that organization. They should be functionality for them to create other users within their organization. The roles are 'admin', 'finance', 'legal', ' support'. The multitenant support should not use the tenantsubdomain.example.com pattern. Each organization should be able to create a 50/50 raffle. 

# Licensing
Each raffle requires a license. The license is granted by a jurisdiction that has a specific geographical location specified by GEOJSON. Each license may have specific requirements for the raffle to abide by... these are things such as "Ticket purchasers must be over 19", "Tickets can only be sold within the Jurisdiction", "2.35% of the total prize fund is required to be paid for the raffle license". Each license is either for a single draw at a specific time/place or may be a recurring draw based on a schedule e.g. weekly. Draws can only be created within the timeframe of the license. Each draw has a start/end datetime for when tickets for that draw can be sold.

# Ticket pricing 
Tickets for the raffle can be sold via various price tiers e.g. 1 Ticket = $5, 3 Tickets = $10

# Draws
Once a draw is closed, no more tickets can be sold for that draw and if the the raffle is a recurring series the next draw is scheduled.

Each draw has a prize pool. The prize pool can consist of 1 main winner and multiple other prizes e.g. "Second place" or "Early bird - prize pack". Once a draw is closed, admins from the organization running the raffle can use a random number generator to pick an eligible ticket from the draw for each prize. If a ticket wins a prize it is ineligible to win any of the other prizes available for that draw. The main prize is typically a 50% of the revenue from the ticket sales from that draw. The remainder of the funds are held for the organization and platform fees, mandatory fees etc are taken from the remainder.

Once the prizes are awarded, the winner is notified via email/sms. The funds are then dispersed to the beneficiaries.

# Ticket purchasers
An end user can create an account that can purchase tickets from any draw in the system. They should be able to log into their account and see all of the tickets that they have purchased and upcoming draws that may interest them.

## Technical Stack
- Rails 8
- PostgreSQL with PostGIS extension
- Multitenancy using https://github.com/erwinm/acts_as_tenant (avoid subdomain pattern)
- UI: DaisyUI components with Hotwire/Turbo/Stimulus
- Money handling: RubyMoney and Money-rails gem with multi-currency support
- Recurring events: ice_cube gem
- Real-time updates: Action Cable for draw notifications
- Use https://github.com/panthomakos/timezone to consider timezone issues, and issues around daylight savings etc.
- Use Stripe Connect
- use Devise gem and Devise passkeys
- Use things like solid_queue instead of dependencies that rely on Redis

## Project Structure Requirements
- Follow Rails 8 conventions with clear separation of tenant-specific logic
- Use concern-based organization for shared multitenancy features
- Separate controllers/views by user type (organization users vs ticket purchasers)
- Implement service objects for complex business logic (draw execution, prize distribution)
